    1. Create `scripts/google-ads/creative-analyser.js` -- the complete, copy-paste ready Google Ads Script:

       CONFIG block at top:
       ```javascript
       var CONFIG = {
         API_ENDPOINT: 'https://YOUR-APP.vercel.app/api/ingest',
         API_KEY: 'YOUR_API_KEY_HERE',
         DATE_RANGE: 'YESTERDAY',
       };
       ```

       main() function:
       - Get customerId from AdsApp.currentAccount().getCustomerId()
       - Get accountName from AdsApp.currentAccount().getName()
       - Get today's date formatted as YYYY-MM-DD
       - Build payload: { accountId: customerId, accountName, date, rsa: [], rsaAssets: [], pmax: [], pmaxAssets: [], display: [], video: [] }
       - Call each query function and populate arrays
       - POST to API_ENDPOINT with x-api-key header
       - Log response code and body
       - If first registration (response includes apiKey), log: "IMPORTANT: Update CONFIG.API_KEY with: {apiKey}"

       Query functions (use GAQL via AdsApp.search() -- NOT the deprecated AdsApp.report()):
       - queryRsaPerformance(): Query ad_group_ad where campaign type is SEARCH and ad type is RESPONSIVE_SEARCH_AD. Select campaign.id/name, ad_group.id/name, ad_group_ad.ad.id, ad_group_ad.ad.responsive_search_ad.headlines, ad_group_ad.ad.responsive_search_ad.descriptions, ad_group_ad.ad_strength, metrics (impressions, clicks, cost_micros, conversions, conversions_value). Filter metrics.impressions > 0.
       - queryRsaAssetPerformance(): Query ad_group_ad_asset_view. Select ad_group_ad, asset, field_type, performance_label, metrics.
       - queryPmaxPerformance(): Query asset_group_asset where campaign type is PERFORMANCE_MAX. NOTE: PMax aggregate metrics (impressions, clicks etc.) are at campaign level, not asset_group level via this view. Query campaign-level metrics separately for PMax using campaign resource.
       - queryPmaxAssetData(): Query asset_group_asset for asset details (type, text, image_url, performance_label).
       - queryVideoPerformance(): Query ad_group_ad where campaign type is VIDEO. Include all standard metrics plus video_views, video_view_rate, average_cpv, quartile rates.
       - queryDisplayPerformance(): Query ad_group_ad where campaign type IN ('DISPLAY', 'DEMAND_GEN'). Include ad_type.

       Helper function collectSearchResults(query) to iterate AdsApp.search() results.

       Helper function to safely extract RSA headlines/descriptions from the search result object (they come as repeated fields in the API, need careful extraction).

       Use `var` instead of `let`/`const` since Google Ads Scripts uses Rhino JS engine (ES5-ish). No arrow functions, no template literals, no destructuring. All functions at top level (no modules/classes).

       Add schedule recommendation comment at top: "Schedule this script to run daily at 3am+ in your account timezone (allows Google Ads stats to finalise)."

    2. Create `scripts/google-ads/README.md` with deployment instructions:
       - Title: "Google Ads Script Setup"
       - Prerequisites: Google Ads account with manager access
       - Step 1: Open Google Ads -> Tools & Settings -> Bulk Actions -> Scripts
       - Step 2: Click "+ New script"
       - Step 3: Paste the entire content of creative-analyser.js
       - Step 4: Update CONFIG.API_ENDPOINT with your app URL
       - Step 5: Run the script once manually (Authorise when prompted)
       - Step 6: Check the script logs for your API key (first run auto-registers)
       - Step 7: Update CONFIG.API_KEY with the key from step 6
       - Step 8: Run again to push with authentication
       - Step 9: Set up daily schedule (3am recommended)
       - Troubleshooting section: common errors and fixes

       - Generates 30 days of historical data (one push per day)
       - Each day includes: 3-5 RSA ads with realistic headlines/descriptions/metrics, 2 PMax asset groups, 3-4 display ads, 2 video ads
       - Metrics should be realistic: impressions in hundreds-thousands, CTR 2-8%, CPA $20-80 range, conversions 1-20 per ad per day
       - Run with: `npx tsx scripts/seed-data.ts`
       - Targets http://localhost:3000/api/ingest by default (configurable via env var)
       - Logs progress: "Pushing day 1/30 for Demo Client A..."
       - Add script to package.json: `"seed": "tsx scripts/seed-data.ts"`
       - Install tsx as dev dependency: `npm install -D tsx`
