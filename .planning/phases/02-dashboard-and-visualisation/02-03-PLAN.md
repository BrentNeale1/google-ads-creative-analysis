**1. Create `components/dashboard/MetricCard.tsx`:**

Presentational component (no "use client" needed if used inside a server component, but add it for flexibility since MetricCards may need interactivity later).

Props:
- `label`: string (metric name)
- `value`: string (pre-formatted value)
- `delta`: `{ value: number; direction: 'up' | 'down' | 'flat' } | null`
- `invertColour`: boolean (default false) -- true for CPA where down = green

Render:
- White card with `bg-white rounded-xl border border-surface-gridline p-5 shadow-sm`
- Label: `text-xs font-medium text-gray-500 uppercase tracking-wide mb-1`
- Value: `text-2xl font-semibold text-gray-900`
- Delta (if not null): flex row with icon + percentage
  - Icon: `ArrowUp` from lucide-react when direction='up', `ArrowDown` when 'down', `Minus` when 'flat'
  - Size 14
  - Colour logic: if invertColour, down=green up=red; otherwise up=green down=red; flat=brand-grey
  - Text: `{delta.value.toFixed(1)}%`
  - Wrapper: `flex items-center gap-1 mt-1 text-sm {colour_class}`
- If delta is null, show nothing (new account with no comparison data)

**2. Create `components/dashboard/MetricCards.tsx`:**

This component receives raw KPI totals for current and comparison periods, computes each metric value and delta, then renders a responsive grid of MetricCards.

Props:
- `current`: `{ impressions: number, clicks: number, costMicros: number, conversions: number, conversionsValue: number }`
- `comparison`: same type, or null if no comparison data
- `accountPrimaryKpi`: `'cpa' | 'roas'` (from account settings, determines which to show)

Implementation:
- Import `METRICS` from `@/lib/constants/metrics` and `convertMicrosToAud` from `@/lib/constants/formatting`
- For each MetricKey in `['impressions', 'clicks', 'ctr', 'conversions', 'cpa', 'roas']`:
  - Compute current value using `METRICS[key].computeFromTotals(current)`
  - Format using `METRICS[key].format(currentValue)`
  - If comparison exists, compute comparison value and delta:
    - `const prev = METRICS[key].computeFromTotals(comparison)`
    - If prev === 0: delta = null
    - Else: `{ value: Math.abs(((curr - prev) / prev) * 100), direction: curr > prev ? 'up' : curr < prev ? 'down' : 'flat' }`
- Render a responsive grid: `grid grid-cols-2 md:grid-cols-3 lg:grid-cols-6 gap-4`
- Render one MetricCard per metric

**3. Create `components/dashboard/ChartTooltip.tsx`:**

"use client" component for custom Recharts tooltip. Per user decision (CONTEXT.md): "Rich tooltips on hover showing date, metric value, **and percentage of total**, formatted to AU locale".

Props: standard Recharts tooltip props (`active`, `payload`, `label`) plus:
- `formatValue: (v: number) => string` -- formatter for the metric value
- `total: number` -- the sum of all values in the dataset, used to compute percentage of total

The parent chart components (TimeSeriesChart and CreativeBarChart) MUST compute `total` as the sum of all `value` fields in the `data` array and pass it to ChartTooltip. For example: `const total = data.reduce((sum, d) => sum + d.value, 0)`.

Render (only when active and payload exists):
- Container: `bg-white rounded-lg shadow-lg border border-surface-gridline p-3`
- Date line: `text-xs text-gray-500 mb-1` -- format label as date using `new Date(label).toLocaleDateString('en-AU', { weekday: 'short', day: 'numeric', month: 'short' })`
- Value line: `text-sm font-semibold` -- use `formatValue(payload[0].value)`
- **Percentage of total line:** `text-xs text-gray-500 mt-0.5` -- compute `(payload[0].value / total * 100).toFixed(1)` and display as e.g. "12.3% of total". Guard against total === 0 (show "—" or skip the line). Format the percentage with `toLocaleString('en-AU')` for AU locale consistency.
- If there are multiple series, show each with its colour dot, value, and percentage of total

Return null when !active or !payload?.length.
