**1. Create `components/dashboard/PerformanceTable.tsx`:**

"use client" component for the sortable, paginated creative performance table.

**Props:**
- `data`: `Array<{ name: string; adId: string; type: string; impressions: number; clicks: number; costMicros: number; conversions: number; conversionsValue: number }>`

**State:**
- `sortKey`: string (default: 'conversions') -- which column is sorted
- `sortDir`: 'asc' | 'desc' (default: 'desc')
- `page`: number (default: 0)

**Constants:**
- `PAGE_SIZE = 25`

**Columns definition:**
Define an array of column configs:
```
{ key: 'name', label: 'Creative', align: 'left', format: (v) => v }
{ key: 'type', label: 'Type', align: 'left', format: (v) => v }
{ key: 'impressions', label: 'Impr.', align: 'right', format: formatNumber }
{ key: 'clicks', label: 'Clicks', align: 'right', format: formatNumber }
{ key: 'ctr', label: 'CTR', align: 'right', format: formatPercentage, compute: (row) => row.clicks / (row.impressions || 1) }
{ key: 'conversions', label: 'Conv.', align: 'right', format: (v) => v.toFixed(1) }
{ key: 'cpa', label: 'CPA', align: 'right', format: formatCurrency, compute: (row) => convertMicrosToAud(row.costMicros) / (row.conversions || 1) }
{ key: 'roas', label: 'ROAS', align: 'right', format: (v) => v.toFixed(2) + 'x', compute: (row) => row.conversionsValue / (convertMicrosToAud(row.costMicros) || 1) }
```

**Sort logic:**
- Compute derived fields (CTR, CPA, ROAS) for each row once
- Sort by `sortKey` in `sortDir` direction
- Handle null/NaN values (push to end)
- On header click: if same key, toggle direction; if different key, set desc

**Pagination:**
- `const totalPages = Math.ceil(sortedData.length / PAGE_SIZE)`
- `const pageData = sortedData.slice(page * PAGE_SIZE, (page + 1) * PAGE_SIZE)`
- Show pagination controls: "Page 1 of N", Previous/Next buttons
- Disable Previous on page 0, Next on last page

**Row highlighting (per CLAUDE.md table rules):**
- Identify the top performer (highest value of current sort column) and worst performer (lowest value)
- Top performer row: `bg-[#E6F4EA]` (surface.rowPositive)
- Worst performer row: `bg-[#FCE8E6]` (surface.rowNegative)
- For CPA, invert: lowest CPA = top (green), highest CPA = worst (red)
- Only highlight when sorting by a metric column (not name/type)

**Table styling (per CLAUDE.md table rules):**
- Header: `bg-[#F1F3F4] text-xs font-semibold uppercase tracking-wider` (surface.tableHeader)
- Header cells: `px-4 py-3 cursor-pointer hover:bg-surface-gridline` with sort indicator arrow
- Body cells: `px-4 py-3 text-sm`
- Numbers: `text-right`, text: `text-left`
- Rows: `border-b border-surface-gridline hover:bg-surface-background`
- Table: `w-full border-collapse`
- Wrapper: `bg-white rounded-xl border border-surface-gridline overflow-hidden shadow-sm`

**Empty state:** If data is empty, show "No creative data for the selected filters" centred in the table area.
