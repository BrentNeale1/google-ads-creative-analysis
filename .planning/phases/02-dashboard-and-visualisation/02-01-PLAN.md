**1. Create `app/dashboard/searchParams.ts`:**
Define a nuqs search params cache using `createSearchParamsCache` from `nuqs/server`:
- `account`: `parseAsString.withDefault('')` -- account ID
- `range`: `parseAsStringLiteral(['7d', '30d', '90d', 'custom'] as const).withDefault('30d')` -- date preset
- `from`: `parseAsIsoDate` -- custom range start (nullable)
- `to`: `parseAsIsoDate` -- custom range end (nullable)
- `campaign`: `parseAsString` -- campaign ID filter (nullable)
- `adGroup`: `parseAsString` -- ad group ID filter (nullable)

Export the cache as `dashboardSearchParams`.

Also export a `resolveDateRange` helper function that takes the parsed params and returns `{ dateFrom: string, dateTo: string }` as YYYY-MM-DD strings:
- For '7d': last 7 days from today
- For '30d': last 30 days from today
- For '90d': last 90 days from today
- For 'custom': use from/to params, fallback to 30d if either is null
Use `date-fns` `subDays` and `format` for date arithmetic. Import `startOfDay` from date-fns to ensure consistent day boundaries.

**2. Create `lib/constants/metrics.ts`:**
Define a `MetricKey` type union: `'impressions' | 'clicks' | 'ctr' | 'conversions' | 'cpa' | 'roas'`.

Export a `METRICS` record mapping each MetricKey to:
- `label`: Display name (e.g., "Impressions", "CTR", "CPA", "ROAS")
- `format`: Formatter function from `lib/constants/formatting.ts` (e.g., `formatNumber` for impressions/clicks, `formatPercentage` for CTR, `formatCurrency` for CPA, `formatCurrencyCompact` for chart axis)
- `formatCompact`: Compact formatter for chart axes (e.g., `formatNumberCompact`, `formatCurrencyCompact`)
- `invertColour`: boolean -- true for CPA (down = good), false for others except ROAS (up = good, same as default)
- `computeFromTotals`: function `(totals: { impressions, clicks, costMicros, conversions, conversionsValue }) => number` that computes the metric value from raw aggregation totals. For CTR: clicks/impressions. For CPA: convertMicrosToAud(costMicros)/conversions. For ROAS: conversionsValue/convertMicrosToAud(costMicros). Guard against division by zero (return 0).

**3. Create `lib/queries/dashboard.ts`:**
This is a server-only module. Add `import 'server-only'` at the top (or use a comment if server-only package isn't installed -- check first).

Import `{ db }` from `@/lib/db`, `* as schema` from `@/lib/db/schema`, and drizzle operators (`eq`, `and`, `gte`, `lte`, `sum`, `sql`).

Export these functions:

a) `fetchKpiMetrics(accountId, dateFrom, dateTo, campaignId?, adGroupId?)` -- Returns `{ impressions, clicks, costMicros, conversions, conversionsValue }` aggregated across ALL 4 daily tables (rsaDaily, pmaxAssetGroupDaily, displayDaily, videoDaily). Use `Promise.all` to query all 4 tables in parallel, then sum the results. Apply campaignId/adGroupId WHERE conditions when provided (note: PMax tables use campaignId but not adGroupId in the same way -- for pmaxAssetGroupDaily, filter by campaignId only). Chain `.mapWith(Number)` on all `sum()` calls. Handle null results (no data) by defaulting to 0.

b) `fetchTimeSeries(accountId, dateFrom, dateTo, campaignId?, adGroupId?)` -- Returns `Array<{ date: string, impressions: number, clicks: number, costMicros: number, conversions: number, conversionsValue: number }>`. Query all 4 tables grouped by date, sum across tables per date. Return sorted by date ascending. This gives the dashboard all metrics per day so the chart can switch metric client-side without re-fetching.

c) `fetchCreativeComparison(accountId, dateFrom, dateTo, campaignId?, adGroupId?)` -- Returns `Array<{ name: string, adId: string, type: string, impressions: number, clicks: number, costMicros: number, conversions: number, conversionsValue: number }>`. Query each table for per-creative totals: RSA grouped by adId (name = adGroupName + " / Ad " + adId), PMax grouped by assetGroupId (name = assetGroupName), Display grouped by adId (name = adName or adId), Video grouped by adId (name = adName or adId). Use `sql<string>` for concatenation. Combine results into a single array.

d) `fetchFilterOptions(accountId)` -- Returns `{ campaigns: Array<{ id: string, name: string }>, adGroups: Array<{ id: string, name: string, campaignId: string }> }`. Query distinct campaign/ad group pairs from rsaDaily + displayDaily + videoDaily for the given account. This powers the cascading filter dropdowns. PMax campaigns use asset groups not ad groups, so include PMax campaigns but with no ad groups.

For all queries: build conditions array with accountId + date range as base, then push campaignId/adGroupId conditions if provided.
