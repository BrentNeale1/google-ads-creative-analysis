**RSA page (app/rsa/page.tsx):**
Async Server Component that orchestrates the entire RSA analysis pipeline:

1. Parse search params using rsaSearchParams cache
2. If no account selected, show a message directing user to select an account from the sidebar
3. Fetch account record to get `primaryKpi` setting
4. Fetch data in parallel using Promise.all:
   ```typescript
   const [creatives, assets, combinations, portfolioAvg, filterOptions] = await Promise.all([
     fetchRsaCreatives(accountId, dateFrom, dateTo, campaignId, adGroupId),
     fetchRsaAssets(accountId, dateFrom, dateTo),
     fetchRsaCombinations(accountId, dateFrom, dateTo),
     fetchRsaPortfolioAvg(accountId, dateFrom, dateTo),
     fetchFilterOptions(accountId, dateFrom, dateTo),  // reuse from dashboard queries
   ]);
   ```
5. **CRITICAL -- Enrich creatives with headline text before tier classification:**
   The analysis engine's `classifyTiers` accepts `CreativeInput[]` with an optional `headlineText` field.
   `RsaCreativeAggregated` from the query layer has a `headlines` field (JSONB array of headline strings).
   Map the query result to `CreativeInput[]` by joining headlines into a single string:
   ```typescript
   const creativeInputs = creatives.map(c => ({
     adId: c.adId,
     kpiValue: primaryKpi === 'cpa' ? c.cpa : c.roas,
     headlineText: (c.headlines as string[])?.join(' | ') ?? '',
     // pass through raw metrics needed by UI components:
     impressions: c.impressions,
     clicks: c.clicks,
     ctr: c.ctr,
     conversions: c.conversions,
     cpa: c.cpa,
     roas: c.roas,
     campaignName: c.campaignName,
     adGroupName: c.adGroupName,
     adStrength: c.adStrength,
   }));
   ```
   This enrichment step is essential because `detectPatterns` reads `TieredCreative.headlineText` to classify copy themes. Without it, pattern detection would have no text to analyse.

6. Run analysis pipeline with enriched data:
   ```typescript
   const tiered = classifyTiers(creativeInputs, primaryKpi);
   const diagnosed = diagnoseUnderperformers(tiered, portfolioAvg);
   const patterns = detectPatterns(tiered);  // reads headlineText from each TieredCreative
   const recommendations = generateRecommendations(tiered, diagnosed, patterns);
   const insightTitle = generateInsightTitle(patterns, overallAvgKpi, primaryKpi);
   ```

7. Render page layout:
   - Page title "RSA Analysis" with subtitle showing account name and date range
   - Reuse FilterBar from dashboard (import and render with RSA-specific search params) for date range and campaign/adGroup filtering
   - Tab navigation (Overview | Assets | Recommendations) using the `tab` search param
   - Overview tab: TierOverview + CreativeLeaderboard + PatternCharts
   - Assets tab: AssetPerformance + CombinationReport
   - Recommendations tab: UnderperformerPanel + RecommendationList

Use the same page structure pattern as dashboard/page.tsx. Server Component at top, client components receive pre-computed data as props.
